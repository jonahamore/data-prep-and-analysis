# Сьома лабораторна робота

## Опис завдання

На основі шостої лабораторної роботи оновити програму, додавши інтелектуальний модуль детекції облич у реальному часі за допомогою глибоких нейромереж (DNN) та винести цей модуль в окремий фоновий потік.

Основні етапи роботи:

* Створення модульної архітектури детектора:
  * Реалізувати окремий клас `FaceDetector`.
  * Завантаження претренованої Caffe-моделі (ResNet-10 SSD).
* Багатопотокова обробка кадрів:
  * Ініціалізація фонового робочого потоку всередині `FaceDetector`.
  * Реалізація асинхронного циклу обробки з опитуванням стану нового кадру для аналізу без блокування GUI-потоку.
  * Забезпечення потокобезпечного обміну кадрами та результатами детекції за допомогою м'ютексів (`std::mutex`, `std::lock_guard`) і прапорця `std::atomic<bool>`.
* Додавання нового режиму обробки зображення — детекція облич (клавіша `F`).

## Опис архітектури

Програма розділена на класи, кожен у власних файлах `.hpp` та `.cpp`:

* `CameraProvider` — отримання кадрів з камери (`cv::VideoCapture`).
* `Display` — створення вікна та виведення кадрів (`cv::imshow`).
* `KeyProcessor` — обробка натискань клавіатури та зберігання поточного режиму (`enum Mode`).
* `FrameProcessor` — обробка кадру залежно від режиму, малювання HUD та рамок облич.
* `MouseHandler` — обробка дій миші.
* `FaceDetector` — детекція облич у фоновому потоці.

`FaceDetector` працює так. У головному потоці метод `submitFrame()` під м'ютексом кладе копію свіжого кадру в спільну змінну. Окремий робочий потік у циклі `workerLoop()` перевіряє, чи з'явився новий кадр, і якщо так — виконує інференс мережі (`net.forward()`) та оновлює координати знайдених облич, теж під м'ютексом. Головний потік методом `getFaces()` забирає останні відомі координати й малює їх на поточному кадрі. Зупинку потоку контролює прапорець `std::atomic<bool> running_`.

Завдяки цьому важка детекція не блокує цикл відображення: відео залишається плавним, а рамка облич може трохи відставати від різких рухів.

## Вимоги до системи та заліза

* ОС: Linux на базі Debian. Роботу виконано та протестовано на **Kali Linux** (працює також на Ubuntu).
* Компілятор з підтримкою C++17 (g++), CMake, make.
* Бібліотека OpenCV з модулем `dnn` (`libopencv-dev`, версія 4.x).
* Камера: фізична вебкамера або віртуальна камера. У цій роботі використовується **DroidCam**, який транслює зображення з камери телефона на комп'ютер.

## Інструкція із запуску

### Linux (Kali / Ubuntu)

```bash
git clone https://github.com/Jonahamore/data-prep-and-analysis
cd data-prep-and-analysis/lab7
```

Перед запуском потрібно надати права на виконання скриптам:
```bash
chmod +x preinstall.sh build.sh run.sh
```
Після цього встановити залежності та завантажити модель детектора:
```bash
./preinstall.sh
```
Зібрати проєкт:
```bash
./build.sh
```
Запустити програму:
```bash
./run.sh
```

Скрипт `preinstall.sh` додатково завантажує два файли нейромережі у папку `assets`:
* `deploy.prototxt` — архітектура мережі;
* `res10_300x300_ssd_iter_140000.caffemodel` — ваги.

### Використання DroidCam

DroidCam створює віртуальний відеопристрій (через модуль `v4l2loopback`), який зазвичай **не** є `/dev/video0`. Тому індекс камери можна передати аргументом скрипту запуску.

1. Запустіть DroidCam-клієнт на Kali та підключіть телефон.
2. Перегляньте список відеопристроїв:
```bash
v4l2-ctl --list-devices
```
3. Знайдіть пристрій DroidCam, подивіться його номер (наприклад `/dev/video2`) та запустіть програму з цим індексом:
```bash
./run.sh 2
```

Якщо аргумент не передано, використовується індекс `0`.
